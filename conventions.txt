If you are contributing to this project, please follow the conventions listed below.


# Code
-) Use spaces for tabs and a tab width of 2 spaces
-) For variable and function names, use lower case and separate words by an underscore, for example: "x", "get_item", "test_value"


# Comments
-) Do not manually(!) write multi-line comments. Such comments are difficult to edit and reformat.
-) Instead, use a text editor that supports automatic word wrapping, for example: Eclipse (Word-Wrap plugin), RStudio (Options/Code/Editing/Softwrap), UltraEdit (Configuration/Editor/Word_Wrap), or Notepad++ (View/Word_Wrap)


# Rd documentation
-) Use 'An object of class \code{"class"}' or 'A \code{"class"} object' to refer to objects of a certain class. For example: 'An object of class \code{"lm"}' or 'An \code{"lm"} object'
-) Describe the '...' (ellipsis) function argument as "\dots arguments passed to or from methods."
